CSS gave us lots of new tools that we could use for layout! No more tables, but none of them were real layout tools. So we abused them.

display: flex;

Flexbox is another display property, much like block, inline-block, or table. Except it was actually meant to be used for layout!

Support

Flexbox has been in the works for about 6 yrs now, and is supported by over 90% of browsers world wide. IE9 is the hangup, but CANIUSE puts IE9 at just 1.5% usage, so there isn't much keeping us from using Flexbox right now.

3 Working Versions

2009 old - display: box;
Chrome 4, Firefox 2, Safari 3.1
2011 tweener - display: flexbox;
Chrome 21, IE 10
2012 current - display: flex;
Chrome 31, Firefox 22, IE 11, Safari 6.1, Opera 12.1

Been in development since the days of Firefox 2 and Chrome 4, it has undergone some changes.
There are 3 versions of the syntax that you might encounter. The first one, Display: Box is pretty much defunct at this point.
Display: Flexbox is often referred to as the TWEENER syntax. It is important to keep it in mind, since it is supported by IE10.
The current version, Display: Flex has been pretty stable since 2012, and is supported by all current browsers.

Flexbox uses a Parent - Child model. The parent "Flex container" contains the child "Flex Items". So the atributes of the "Flex container" effect all the "Flex Items", but each "Flex Item" can also have its own individual attributes.

Family Values

"Parent" Flex Container

display: flex;
flex-direction: row;
flex-wrap: nowrap;
justify-content: flex-start;
align-items: stretch;
align-content: stretch;

The parent containers properties only effect its direct children. These are the defaults and are kind of broad strokes. I am going to run through them quickly, so we can get to the samples which are more interesting.

display: flex;

Parent is a block

display: inline-flex;

Parent is a inline-block

Display: Flex is the property that puts things into motion. But there are actually 2 options for the display property. Both effect the flex children the same. Flex is the most commonly used, and makes the parent behaive like a block. Inline-flex behaives like an inline block. This is the only attribute that has any effect on the parennt container.

Controls direction of children on main axis. Horizontal flow.

Simply adding Display: flex puts things into action and implements those defaults.
The first one we will look at is Flex-Direction, which defaults to Row.
Flexbox uses the concept of a Main Axis and a Cross Axis.
By default the Main Axis runs horizontally, Left to Right. And the Cross Axis Runs Top to Bottom. Row lines the items up along that Horizontal Main Axis.

Swaps Main Axis and Cross Axis. Vertical flow.

Changing the Flex-Direction to Column Changes the Main Axis to Vertical Top to Bottom, and the Cross Axis to Left ot Right.
